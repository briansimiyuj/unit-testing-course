Test Driven Development is a process that writing the test before the code. It is a way to ensure that the code is working as expected and that it is not breaking any existing functionality.

Let's create a function that creates cards, it takes two parameters, the first one is the name of the card and the second one is the value of the card. 

We now create a test for this function (refer to `createCards.test.js`). Once we have the test, we can start writing the function. (refer to `createCards.js`)
